# ASMV-19: Homepage wireframe v2: statement footer + larger hero image containers

Adjust the homepage wireframe based on live review feedback: make the front page feel like a large image-led area, represent CTA buttons clearly, and remove the moving header behavior for review clarity.

### Scope
- Enlarge hero/image placeholder containers and increase first-screen visual dominance.
- Add explicit button representations in the hero section.
- Replace hide-on-scroll/fixed header behavior with a static header.
- Upgrade footer to a stronger statement-driven composition.

### Files
- `src/app/page.tsx`
- `src/components/global/Header.tsx`
- `src/components/global/Footer.tsx`

### Acceptance Criteria
- Front page reads as a large hero zone rather than a small intro block.
- CTA/button affordances are visible in first viewport.
- Header no longer shifts/hides with scroll.
- Footer delivers a clear closing statement with stronger visual weight.
